# 连锁眼镜店验光转化分析系统

## 项目简介

本系统基于 Python、Pandas 和 Plotly (Dash) 构建，针对连锁眼镜店业务进行全面的验光转化数据分析。系统支持从验光到配镜的全链路转化追踪、门店及验光师绩效对比、镜片品类矩阵分析、异常订单识别、售后关联分析等核心功能。

## 技术栈

- **Python 3.11**
- **Pandas 2.2** - 数据清洗与计算
- **Plotly 5.22** - 交互式数据可视化
- **Dash 2.17** - Web 交互式分析仪表盘
- **Dash Bootstrap Components** - UI 样式组件

## 原始需求

> 连锁眼镜店验光转化分析需要用 Python、Pandas 和 Plotly 完成，输入门店、验光师、顾客年龄、验光项目、处方结果、镜片推荐、试戴、报价、成交、复购和售后数据。分析要清洗同一顾客多次到店、跨店购买、处方缺失、退货、赠品订单和售后返修，计算验光到配镜转化、客单价、镜片升级率、儿童近视防控接受度、验光师差异和门店服务影响。输出门店对比、验光师漏斗、品类矩阵、异常订单、售后关联和指标定义，筛选年龄段或镜片类型后明细必须同步。

> 扩展多次到店归因：同一顾客先验光、隔日成交、跨店购买或线上补单时，分析按规则归到合适门店和验光师，并展示不同归因口径对转化率的影响。
> 扩展镜片推荐效果：防蓝光、渐进、多焦点、儿童控制和高折射镜片的推荐率、接受率、退货率和售后反馈要分开看，图表按年龄、处方度数和价格带联动。

## 启动方式

### 前置要求

- Python 3.9 及以上版本
- pip 包管理器
- （可选）Docker & Docker Compose

### 启动步骤

#### 1. 安装依赖

```bash
pip install -r requirements.txt
```

#### 2. 启动服务

```bash
python app.py
```

访问地址：**http://localhost:8050**

首次启动会自动生成模拟数据（包含 6 家门店、12 名验光师、约 2000 名顾客、3500 条验光记录），数据文件保存在 `data/` 目录下。

### Docker 一键启动（推荐）

#### 1. 构建并启动

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

访问地址：**http://localhost:8050**

#### 2. 停止和清理服务

```bash
docker compose down
```

## 核心功能模块

| 模块 | 功能说明 |
|------|---------|
| 总览看板 | 核心 KPI 指标、整体转化漏斗、年龄段综合分析、数据质量概览 |
| 门店对比 | 各门店验光数/成交数/转化率/营收/客单价多维度对比 |
| 验光师分析 | 验光师转化漏斗对比、绩效指标明细 |

| 售后关联 | 售后率与客单价关联分析、返修/投诉分布 |
| 指标定义 | 全部分析指标的业务定义说明 |

## 数据清洗规则

系统自动执行以下数据清洗逻辑：

1. **同一顾客多次到店**：标记复购顾客，统计到店次数、首次/末次到店日期
2. **跨店购买识别**：识别同一顾客在不同门店的消费记录
3. **处方缺失处理**：标记未完成验光、无处方成交等异常情况
4. **退货订单**：识别有效退货（已成交后退货），排除无效退货记录
5. **赠品订单**：标记赠品订单，在计算客单价和营收时剔除
6. **售后返修**：分类标记返修、投诉、调整等售后类型，识别负面售后

## 核心指标

- 验光成交转化率
- 客单价 (AOV)
- 镜片升级率（高端/防控镜片接受率）
- 儿童近视防控接受度
- 复购率
- 售后率 / 返修率 / 投诉率
- 推荐率 / 接受率 / 退货率 / 售后率（镜片推荐效果）
- 首次触点归因 / 末次触点归因 / 线性归因 / 时间衰减归因（多口径归因）
- 跨店成交率 / 线上补单率

## 交互式筛选

系统顶部提供三维联动筛选：
- **年龄段筛选**：0-12岁、13-18岁、19-30岁、31-45岁、46-60岁、60岁以上
- **镜片品类筛选**：基础、功能、高端、防控
- **门店筛选**：全部 6 家门店

筛选后所有图表和明细数据表同步更新。
